## Overview

### The Vision

Finite currently lets users reflect on their past (Chapters, Quality ratings). But goal-oriented users need to see their **future** on the same grid. Horizons transforms Finite from a reflection app into a **life planning system** by letting users pin milestones to future weeks.

**The insight:** Mortality IS the ultimate deadline. When you visualize a goal on your life grid, you're anchoring it to your finite existence. This creates "temporal scarcity" that dramatically increases commitment.

### Milestone Status

```swift
extension Milestone {
    enum Status {
        case upcoming      // Target week is in future
        case thisWeek      // Target week is current week
        case overdue       // Target week passed, not completed
        case completed     // User marked complete
    }
    
    func status(currentWeek: Int) -> Status {
        if isCompleted { return .completed }
        if targetWeekNumber < currentWeek { return .overdue }
        if targetWeekNumber == currentWeek { return .thisWeek }
        return .upcoming
    }
    
    func weeksRemaining(from currentWeek: Int) -> Int {
        max(0, targetWeekNumber - currentWeek)
    }
    
    func targetAge(birthDate: Date) -> Int {
        targetWeekNumber / 52
    }
}
```

### Query Pattern (Critical for Bug Fix)

```swift
// In your view or view model
// Query ALL milestones - this returns an array, not a single object
@Query(sort: \Milestone.targetWeekNumber) 
private var allMilestones: [Milestone]

// Computed properties for filtering
private var upcomingMilestones: [Milestone] {
    allMilestones.filter { !$0.isCompleted && $0.targetWeekNumber >= currentWeekNumber }
}

private var nextMilestone: Milestone? {
    upcomingMilestones.first
}

private var overdueMilestones: [Milestone] {
    allMilestones.filter { !$0.isCompleted && $0.targetWeekNumber < currentWeekNumber }
}

private var completedMilestones: [Milestone] {
    allMilestones.filter { $0.isCompleted }
}

// Lookup for grid rendering - O(1) access
private var milestonesByWeek: [Int: Milestone] {
    Dictionary(uniqueKeysWithValues: 
        upcomingMilestones.map { ($0.targetWeekNumber, $0) }
    )
}
```

## Grid Visual Treatment

### Color Specification

```swift
// Horizons mode color function
func horizonsColor(for weekNumber: Int, milestone: Milestone?) -> Color {
    // Milestone weeks get special treatment
    if let milestone = milestone {
        switch milestone.status(currentWeek: currentWeekNumber) {
        case .upcoming, .thisWeek:
            return milestone.category?.color ?? Color("text-primary")
        case .overdue:
            return Color.red.opacity(0.8)
        case .completed:
            return Color("text-secondary").opacity(0.5)
        }
    }
    
    // Non-milestone weeks
    if weekNumber < currentWeekNumber {
        return Color("week-filled").opacity(0.3)  // Dimmed past
    } else if weekNumber == currentWeekNumber {
        return Color("week-current")
    } else {
        return Color("week-empty")
    }
}
```

### Builder State Initialization (Critical for Bug Fix)

```swift
struct MilestoneBuilderView: View {
    let mode: MilestoneBuilderMode
    let user: User
    
    // Form state - initialized based on mode
    @State private var name: String
    @State private var targetWeekNumber: Int
    @State private var category: String?
    @State private var notes: String
    
    init(mode: MilestoneBuilderMode, user: User, onSave: (() -> Void)?, onDelete: (() -> Void)?) {
        self.mode = mode
        self.user = user
        
        // CRITICAL: Initialize state correctly based on mode
        switch mode {
        case .add(let preselectedWeek):
            // FRESH state for new milestone
            _name = State(initialValue: "")
            _targetWeekNumber = State(initialValue: preselectedWeek ?? (user.currentWeekNumber + 52))
            _category = State(initialValue: nil)
            _notes = State(initialValue: "")
            
        case .edit(let milestone):
            // PRE-FILL from existing milestone
            _name = State(initialValue: milestone.name)
            _targetWeekNumber = State(initialValue: milestone.targetWeekNumber)
            _category = State(initialValue: milestone.category)
            _notes = State(initialValue: milestone.notes ?? "")
        }
    }
    
    private func save() {
        switch mode {
        case .add:
            // âœ… CREATE NEW
            let milestone = Milestone(
                name: name.trimmingCharacters(in: .whitespaces),
                targetWeekNumber: targetWeekNumber,
                category: category
            )
            milestone.notes = notes.isEmpty ? nil : notes
            modelContext.insert(milestone)  // â† INSERT
            
        case .edit(let milestone):
            // âœ… UPDATE EXISTING
            milestone.name = name.trimmingCharacters(in: .whitespaces)
            milestone.targetWeekNumber = targetWeekNumber
            milestone.category = category
            milestone.notes = notes.isEmpty ? nil : notes
            milestone.updatedAt = Date()
            // No insert needed, SwiftData tracks changes
        }
        
        try? modelContext.save()
        dismiss()
    }
}
```

## Edge Cases

### 1. Milestone on Current Week

**Scenario:** User sets milestone for this week, or time passes and milestone is now "this week".

**Behavior:**
- Status = `.thisWeek`
- Marker shows with slight highlight/glow
- Context bar shows "This week" instead of "X weeks"
- Prompt: "Did you complete this?" (optional)

### 2. Multiple Milestones on Same Week

**Scenario:** User creates "Turn 30" and "Run marathon" both for week 1590.

**Behavior:**
- Grid shows single marker with count badge (`â¬¡Â²`)
- Tapping marker opens list sheet filtered to that week
- Context bar shows first one (alphabetically or by creation)
- Both appear in full list sheet

### 3. Overdue Milestone (Target Week Passed)

**Scenario:** Milestone was set for week 1580, current week is 1582.

**Behavior:**
- Status = `.overdue`
- Marker color: Red tint
- Marker position: Still at week 1580 (in the "past" zone)
- Context bar: Shows in separate "Overdue" indicator
- List sheet: Separate "Overdue" section at top
- Detail sheet: Shows "Overdue by X weeks", options to complete or reschedule

### 4. Far Future Milestone (Beyond Life Expectancy)

**Scenario:** User sets milestone at age 120 (week 6240).

**Behavior:**
- Allow it (user's choice)
- No warning (mortality awareness is the app's theme)
- Marker appears on grid (if grid extends that far)
- May not be visible without scrolling/zooming (future consideration)

### 5. Milestone at Edge of Grid

**Scenario:** Milestone is at week 4160 (age 80), grid ends at 4160.

**Behavior:**
- Marker appears at last visible position
- If milestone is beyond visible grid, show indicator
- Consider: "X milestones beyond visible range" note

### 6. Rapid Add/Delete

**Scenario:** User quickly adds 5 milestones then deletes 3.

**Behavior:**
- Each operation saves immediately
- SwiftData handles concurrency
- UI updates reactively via @Query
- Haptic feedback on each action

### 7. Empty Name Validation

**Scenario:** User tries to save milestone with blank name.

**Behavior:**
- "Set Horizon" / "Save" button disabled
- Visual indication on text field (optional subtle highlight)
- No error message needed (button state is clear)

### 8. Milestone Becomes "This Week"

**Scenario:** App opens on Monday, milestone was for "this week".

**Behavior:**
- On app launch, check for `.thisWeek` milestones
- Show subtle notification: "This week: Run marathon"
- Context bar highlights it prominently
- Consider notification (future: push notification)
